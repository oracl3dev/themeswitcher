
# themeswitcher
Wallpaper changer script + insta applying newly generated wal colors (swww + wal)

# Software Dependencies #
Bash	The scripts are written in Bash.
swww	Wallpaper daemon for Hyprland (swww-daemon). Handles wallpaper setting and transitions.
pywal	Generates color schemes from wallpapers, outputs colors for terminal and apps.
kitty	Terminal emulator. Needs to read wal-generated colors.
wofi	Menu launcher for picking themes or wallpapers manually.

## Optional but recommended:

Tool	Purpose
Hyprland	Wayland compositor where wallpapers and colors are applied.
kill/pgrep	Built-in Linux commands used to manage processes (swww-daemon, kitty).





# Config Requirements #

Kitty config must include the wal colors file:

include ~/.cache/wal/colors-kitty.conf

Ensures new Kitty terminals automatically inherit the colors generated by pywal.

Theme configuration (themes.conf) format:

theme_name|/full/path/to/wallpaper.png

theme_name → displayed in Wofi for selection

wallpaper path → used by swww and pywal

State file location (created automatically):

~/.config/theme/.current_theme

Stores the last applied theme and wallpaper for restore-theme.sh.


# Suggested Folder Structure (works flawlessly like this on my hyprland) #

~/.config/theme/
├── scripts/
│   ├── apply-theme.sh
│   ├── restore-theme.sh
│   └── theme-picker.sh
├── .current_theme      # auto-generated
~/.config/themes/
└── themes.conf         # theme definitions
~/Pictures/wallpapers/  # folder with your wallpaper images


# Permissions # 

All scripts should be executable!

chmod +x ~/.config/theme/scripts/*.sh


# Usage #

 Pick and apply a theme: 

~/.config/theme/scripts/theme-picker.sh

Restore last applied theme:

~/.config/theme/scripts/restore-theme.sh

Manually apply a specific wallpaper:

~/.config/theme/scripts/apply-theme.sh <theme_name> /full/path/to/wallpaper.png
